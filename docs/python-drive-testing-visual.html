What Python asserts now

implementedhardware-in-loop

  • Controller decoding: sticks, triggers, buttons, D-pad.
  • Drive mixer: arcade, tank, D-pad virtual axes, precision, brake.
  • Motor intent: M1/M2 speed, dir, fwd_duty, rev_duty, freq.
  • Manual motor modes: momentary and latching.
  • Servo/ESC pulse intent: S1/S2 pulse_us, duty, arming phase.
  • Battery power behavior: GOOD full, WARN reduced, LOW disabled.
  • Failsafe: disconnect stops drive motors.
Important: this does not measure voltage/waveforms with a scope. It proves the firmware's internal decision and output intent on real hardware.
